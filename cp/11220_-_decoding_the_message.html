11220 - Decoding the message.

View Problem Statement

Problem Summary

Chico and Maria communicate using a secret code: from each line of their message, the first word gives the first letter, the second word the second letter, and so on. If a word is too short, the next word provides the missing letter. At the end of each line, the current decoded word is completed, and decoding continues from the first letter of the first word in the next line. The task is to decode multiple test cases and output each decoded word on a separate line.

Logic Explanation

  1. Read number of test cases
    - First input line contains T (number of test cases).
    - br.readLine() reads T and another br.readLine() skips the blank line before the first case.
  2. Loop through each test case
    - Outer loop for (int i = 1; i <= testcase; i++) processes each test case.
    - Prints Case #i: to mark the start of the case.
  3. Read each line of the test case
    - Inside the while(true) loop, each message line is read until a blank line appears.
    - Blank line indicates the end of the current test case.
  4. Process each line to decode
    - Split the line into words: String[] s = line.split(" ");
    - Keep track of which letter to pick using index.
       • First word → pick its 1st letter (index 0)
       • Second word → pick its 2nd letter (index 1)
       • Third word → pick its 3rd letter (index 2)
    - If a word is too short, skip it and continue to the next word.
    - if (s[j].length() > index) ensures only words long enough are considered.
  5. Output decoded word
    - Each decoded word is printed on its own line.
    - At the end of the test case, a blank line is printed if there are more test cases.

Key Points
- Uses index to track which character position to pick for each word in sequence.
- Handles blank lines to separate test cases.
- Ensures proper formatting (blank line between test cases).
